# Oncall Agent

1. 基于 [deer-flow 2.0](https://github.com/bytedance/deer-flow) 实现这个 Oncall Agent。
2. 这个oncall助手，解决的是生活服务B端商家创建商品时遇到的投诉、咨询问题，比如：
  - 为什么我在创建 `购物>果蔬生鲜>水果的团购商品`时，上品表单中没有‘商家平台商品ID’？
  - 为什么我在创建 `购物>果蔬生鲜>水果的代金券商品` 时，‘券码类型’组件没有‘商家券’这个选项？
3. 支撑商品上单页面的代码有以下这些
  - 商品模板：商品类目、商品类型、商品子类型、模板类型、模板子类型（五元组）决定一张具体的模板，比如‘购物>果蔬生鲜>水果’就是商品类目、‘团购’就是商品类型。可以通过MCP `ace_ai_locate_template` 获取模糊匹配获取具体的五元组信息，如果无法精确定位五元组信息需要interrupt给human确认
  - 商品字段的配置schema信息，示例中的‘商家平台商品ID’、‘券码类型’即schema中的title字段，需要clarity具体的字段，如果无法精确定字段信息需要interrupt给human确认
  - 组件的代码，组件的实现在一个codebase的repo中，需要shallow clone仓库到一个默认的缓存区，通过grep、glob等工具去定位代码实现
  - 还能作用于组件交互的代码，在商品字段的配置schema信息中，schema的properties中有一些代码，在解释器中执行可以作用于组件的表现
4. 有了上面这些信息，你应该就可以解释为什么这个模板中这个组件为什么会出现预期外的交互了
